Implementing light weight Windows Authentication

This article explains how to integrate the out of the box Security Token Service provided in the vanilla deployment (also referred to as ISHSTS) with Active Directory.

Acknowledgements

Content Manager is installed on ish.example.com and ISHSTS is installed on ish.example.com/ISHSTS.

Set the deploymentName variable:

$deploymentName="InfoShare"

Enable Windows Authentication

Configure ISHSTS to provide windows authentication:

# Set ISHSTS to window authentication
Set-ISHSTSConfiguration -ISHDeployment $deploymentName -AuthenticationType "Windows"

Set-ISHSTSConfiguration will raise a warning when the connection combines one all of the following conditions:

  • The integrated database is a SQL Server.
  • The connection string uses an integrated authentication instead of a username/password combination.

The warning is raised because some additional actions are required on the database to complete this configuration.

Configure the SQL Server database

We need to grant permissions to the server and database for the principal that matches the computer account that hosts ISHSTS.

Save-ISHIntegrationDBSTSSQLServerConfiguration will export a sql or PowerShell script based on the value provided to the -Type parameter.

For example:

# Save the powershell script to configure the SQL database
Save-ISHIntegrationDBSTSSQLServerConfiguration -ISHDeployment $deploymentName -Type PS1 -FileName "Invoke-GrantPermissionsOnSQLServer.ps1"

# Save the sql script to run directly on SQL Server
Save-ISHIntegrationDBSTSSQLServerConfiguration -ISHDeployment $deploymentName -Type SQL -FileName "GrantPermissions.sql"

If you saved a SQL script, execute the code above on the same SQL Server referenced by the connection string.

The saved PowerShell script embeds the SQL server name and database and uses windows authentication to access the database. To execute it, there are two conditions:

  • The current user has access to the target database. The script uses windows authentication and for this reason Save-ISHIntegrationDBSTSSQLServerConfiguration generates a warning.

WARNING: the current implementation of .ps1 works only with windows authentication.

  • SQLPS PowerShell module is installed. The module is normally installed along SQL Server Management Studio.

If all conditions are satisfied, executing the script completes the task.

& .\Invoke-GrantPermissionsOnSQLServer.ps1
