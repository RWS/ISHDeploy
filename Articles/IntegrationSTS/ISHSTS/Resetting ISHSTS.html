Resetting ISHSTS

This article explains how to reset the ISHSTS configuration.

ISHSTS stores all configuration in a SQL compact database. When Content Manager is installed, the database is created and automatically seeded.

To help reset ISHSTS, the module offers the Reset-ISHSTS cmdlet:

$deploymentName="InfoShare"
Reset-ISHSTS -ISHDeployment $deploymentName

Reset-ISHSTS is not the same as Undo-ISHDeployment. As a brief comparison:

  • Reset-ISHSTS deletes only the SQL compact database. When the database is rebuilt, all changes applied by other cmdlets are respected, with the exception of Set-ISHSTSRelyingParty. Set-ISHSTSRelyingParty configures ISHSTS for third party integrations, and the configuration information will be lost with the reset. To configure the third party systems, execute the commands with Set-ISHSTSRelyingParty as discussed in Integrating 3rd party service providers with ISHSTS.
  • Undo-ISHDeployment deletes the SQL compact database and reverts all changes applied by other cmdlets. When the database is rebuilt, then only the original vanilla configuration is used.

Reset-ISHSTS offers confidence when adding relying parties to ISHSTS. With Reset-ISHSTS you always remove all extra entries and know that the system will reset itself to the current functioning configuration.
