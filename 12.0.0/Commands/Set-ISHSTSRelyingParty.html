Set-ISHSTSRelyingParty

SYNOPSIS

Sets relying party to infosharests database. The switches acts as filter based on the RP name.

SYNTAX

UNNAMED_PARAMETER_SET_1

Set-ISHSTSRelyingParty -Name <string> -Realm <string> [-LC] -ISHDeployment <ISHDeployment>

UNNAMED_PARAMETER_SET_2

Set-ISHSTSRelyingParty -Name <string> -Realm <string> [-BL] -ISHDeployment <ISHDeployment>

UNNAMED_PARAMETER_SET_3

Set-ISHSTSRelyingParty -Name <string> -Realm <string> [-EncryptingCertificate <string>]
 -ISHDeployment <ISHDeployment>

DESCRIPTION

The Set-ISHSTSRelyingParty cmdlet sets relying party to from the infosharests database.

EXAMPLES

---------- EXAMPLE 1 ----------

PS C:\>Set-ISHSTSRelyingParty -ISHDeployment $deployment -Name "BL: WCF" -Realm "https://realm.example.com/BL/wcf/"

This command sets relying parties to infosharests database. Parameter $deployment is a deployment name or an instance of the Content Manager deployment retrieved from Get-ISHDeployment cmdlet.

As prefixes can be configured for custom handling, it is not possible to change it when updating existing data for relying party.
